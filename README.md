# ACR Hanote

Personal notes and development journal by Hisanobu Araki.

This repository contains writings related to:

- The history and philosophy behind ACR (Across Report)
- Real-world experiences from early IT and retail operations
- Thoughts on system design, printing, and field engineering
- Essays intended for younger generations interested in IT

Most articles are written in Japanese and published via Zenn.  
This repository serves as the source and archive for those writings.

---

## About ACR (Across Report)

ACR is a printer-independent reporting architecture.

Render once. Output everywhere.

It separates layout description, rendering, and output generation,  
allowing the same template to be used across different environments.
